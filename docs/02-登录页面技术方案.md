# 登录页面技术方案

## 1. 技术栈

- Vue 3：页面和组件开发。
- Vite：开发服务和构建。
- TypeScript：类型约束。
- Element Plus：表单、输入框、按钮、提示组件。
- Vue Router：登录页、Dashboard 和路由守卫。
- Pinia：登录态管理。
- Vitest + Vue Test Utils：单元测试和组件测试。
- GitHub Actions：CI 验证。

## 3. 路由设计

| 路径 | 页面 | 权限 |
| :--- | :--- | :--- |
| `/` | 重定向到 `/dashboard` | - |
| `/login` | 登录页 | 仅未登录用户 |
| `/dashboard` | 最小工作台 | 需要登录 |

路由守卫规则：

- `requiresAuth=true` 且未登录：跳转 `/login?redirect=<原路径>`。
- `guestOnly=true` 且已登录：跳转 `/dashboard`。

## 4. 登录态设计

Pinia store 保存：

- `token`
- `user`
- `isAuthenticated`

存储规则：

- 勾选“记住我”：写入 `localStorage`。
- 不勾选“记住我”：写入 `sessionStorage`。
- 退出登录：同时清空 `localStorage` 和 `sessionStorage`。

## 5. Mock API

接口函数：`loginWithPassword(payload)`。

请求参数：

```ts
{
  email: string
  password: string
  rememberMe: boolean
}
```

成功条件：

- `email === "demo@example.com"`
- `password === "Password123!"`

成功返回：

```ts
{
  token: "mock-token-persistent" | "mock-token-session",
  user: {
    email: "demo@example.com",
    name: "Demo User"
  }
}
```

失败行为：

- 抛出错误：`账号或密码错误，请检查后重试。`

## 6. 表单校验

登录表单由 Element Plus `el-form` 承载：

- 邮箱：必填、邮箱格式。
- 密码：必填、最小 8 位。
- 提交时先校验，校验失败不调用 Mock API。
- 调用 Mock API 期间登录按钮显示 loading。

## 7. 错误处理

- 表单格式错误显示在字段下方。
- 凭证错误显示页面级 error alert。
- 未知错误显示“登录失败，请稍后重试。”。

## 8. 测试方案

自动化测试覆盖：

- Mock 登录成功。
- Mock 登录失败。
- 登录态持久化。
- 退出登录清理状态。
- 未登录访问 Dashboard 重定向登录页。
- 已登录访问登录页重定向 Dashboard。
- 登录页空表单校验。
- 登录页成功提交跳转。
- 登录页失败提交展示错误。
- Dashboard 展示用户并支持退出。
