# 登录页面需求文档

## 1. 背景

本项目用于验证蜂鸟系统从 GitHub Issue 到 Claude Code 执行、PR 交付的真实链路。业务需求选择一个范围清晰、可独立验收的 Vue 登录页面 MVP。

第一版登录页只做前端闭环，使用 Mock 登录接口，不接真实后端。

## 2. 目标用户

- 需要登录进入后台或工作台的普通用户。
- 负责验收蜂鸟执行结果的开发者或产品负责人。

## 3. 用户路径

1. 用户访问 `/login`。
2. 用户输入邮箱和密码。
3. 系统校验邮箱格式和密码长度。
4. 用户提交登录。
5. Mock 登录成功后，系统保存 token 和用户信息，并跳转 `/dashboard`。
6. 登录失败时，系统停留在登录页并展示明确错误。
7. 未登录用户访问 `/dashboard` 时，系统重定向到 `/login`。
8. 已登录用户访问 `/login` 时，系统重定向到 `/dashboard`。
9. 用户在 Dashboard 点击退出登录，系统清空登录态并回到 `/login`。

## 4. 功能需求

### 4.1 登录页

- 显示邮箱账号输入框。
- 显示密码输入框。
- 显示“记住我”复选框。
- 显示登录按钮。
- 显示默认演示账号提示。
- 登录提交时按钮进入 loading 状态。

### 4.2 表单校验

- 邮箱必填。
- 邮箱必须符合邮箱格式。
- 密码必填。
- 密码不少于 8 位。

### 4.3 Mock 登录

- 成功账号：`demo@example.com`
- 成功密码：`Password123!`
- 成功后返回 mock token 和用户信息。
- 失败后返回“账号或密码错误，请检查后重试。”。

### 4.4 登录态

- 勾选“记住我”时，登录态保存到 `localStorage`。
- 不勾选“记住我”时，登录态保存到 `sessionStorage`。
- 退出登录时清空两种存储中的登录态。

### 4.5 路由行为

- `/login`：登录页。
- `/dashboard`：受保护的最小占位页。
- `/`：重定向到 `/dashboard`。
- 未登录访问受保护页面时重定向 `/login`。
- 已登录访问 `/login` 时重定向 `/dashboard`。

## 5. 非功能需求

- 页面在桌面和移动视口下都不能出现明显布局错位。
- 表单错误提示必须直接可见。
- 登录态逻辑必须有自动化测试覆盖。
- CI 必须执行 lint、test、build。

## 6. 验收标准

- [ ] 访问 `/login` 可以看到邮箱、密码、记住我、登录按钮。
- [ ] 空表单点击登录时显示必填校验。
- [ ] 输入非法邮箱时显示邮箱格式错误。
- [ ] 输入 `demo@example.com` / `Password123!` 后跳转 `/dashboard`。
- [ ] 输入错误密码时停留登录页并显示错误提示。
- [ ] 未登录直接访问 `/dashboard` 会跳转 `/login`。
- [ ] 已登录访问 `/login` 会跳转 `/dashboard`。
- [ ] Dashboard 显示当前用户邮箱。
- [ ] 点击退出登录后清空登录态并回到 `/login`。
- [ ] `pnpm lint`、`pnpm test`、`pnpm build` 全部通过。

## 7. 非目标

- 不接真实后端。
- 不做注册。
- 不做忘记密码。
- 不做验证码登录。
- 不做第三方登录。
- 不做复杂权限系统。
- 不做国际化和主题系统。
